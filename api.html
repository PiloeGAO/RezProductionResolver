Python API¶

Basic Workflow¶

  1. Initialize database:

    from rezplugins.command.production_resolver_lib import ProductionResolverDatabase

with ProductionResolverDatabase(load_production=False) as db:
    db.initialize()

Warning

Initializing the ProductionResolverDatabase without load_production=False will allow you to edit the production database directly. This is a bad practice because we always want to make changes in a staging area, test them thoroughly, and then deploy to production when everything is validated.

  1. Add packages to a context:

    with ProductionResolverDatabase(load_production=False) as db:
    [...]
    db.add_package(("my_project", "assets", "character"), "maya-2024", software="maya")

  2. Save changes to staging:

    with ProductionResolverDatabase(load_production=False) as db:
    [...]
    db.save(comment="Your modifications.")

  3. Deploy changes to production:

    with ProductionResolverDatabase(load_production=False) as db:
    [...]
    db.deploy()

Common Operations¶

  • Add Package:

    db.add_package(context_tuple, package_name, step=None, software=None)

  • Remove Package:

    db.remove_package(context_tuple, package_name, step=None, software=None)

  • Get Package List:

    packages = db.get_package_list(*context_elements)

Methods and Classes¶

Production Resolver Library.

This module provides a ProductionResolverDatabase class to manage SQLite-based package context configurations for Rez production environments. It supports context-aware package management, validation, and deployment workflows.

class rezplugins.command.production_resolver_lib.HistoryEditOperation(*values)[source]¶

Enumeration of possible operations for history edits.

This enum defines the types of operations that can be recorded in the production resolver’s history. Each value corresponds to a specific action performed on a package context.

INSTALL¶

Represents an installation operation (value 1).

Type:

int

UNINSTALL¶

Represents an uninstallation operation (value 2).

Type:

int

Example

Adds a package to the database under the specified context.

The package is inserted into the ‘package’ table with the given context ID, step, and software. If validation is enabled, it ensures the package is valid using Rez’s ResolvedContext.

  • step (str, optional) – The step this package is associated with.

  • validate (bool, optional) – Whether to validate the package using Rez. Defaults to True.

deploy() → None[source]¶

Deploys the current database to production, optionally creating a backup of the existing production database.

If the ‘keep_history’ configuration is enabled, a timestamped backup of the production database is created. The current database connection is then copied to the production database.

Note

Backup files are saved in the history_folder_path directory with filenames formatted as YY_MM_DD_HH_MM_SS_ffff.db where ffff represents microseconds.

Raises:

  • RuntimeError – If the currently edited database is a production database.

  • sqlite3.OperationalError – If an error occurs during database operations.
